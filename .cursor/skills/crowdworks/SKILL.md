# CrowdWorks 自動化スキル

CrowdWorks の案件検索 → 全案件タブ展開 → 提案文生成 → 応募フォーム入力を browser MCP で実行する。

## トリガー

「クラウドワークスで案件探して」「CWで10件探して」「CWの案件に提案して」等。

## 前提条件

案件詳細ページは `/public/jobs/{id}` の公開 URL であり、**閲覧にログインは不要**。
Phase 1（検索 + タブ展開）はログインなしで実行できる。

**ログイン確認は Phase 2（応募フォーム遷移時）で行う。** Phase 1 では不要。

---

## Phase 1: 一括検索 + タブ展開（最初の1回だけ実行）

### Step 1-1: 検索ページで案件を取得

佐藤の指定に応じて URL を構成し、`browser_navigate` で開く。

**AI カテゴリ直接アクセス（推奨）:**
```
https://crowdworks.jp/public/jobs/group/ai_machine_learning
```
佐藤が「AI案件」「CW案件」と言った場合はこの URL を使う。

サブカテゴリ URL:
- `crowdworks.jp/public/jobs/group/chatgpt_development` — ChatGPT 開発
- `crowdworks.jp/public/jobs/group/ai_chatbot_development` — AI チャットボット

**キーワード検索（特定の技術・分野を指定された場合）:**
```
https://crowdworks.jp/public/jobs/search?keyword={キーワード}&order=new
```

### Step 1-2: フィルタリング

`browser_snapshot` を実行し、`listitem` ロールの要素から案件情報を抽出。

各 `listitem` の `name` 属性に含まれる情報:
- 案件タイトル、カテゴリ名、報酬タイプ・金額、応募数・契約状況、残り日数

子要素の `heading` (level 3) にクリーンなタイトルがある。

**フィルタ基準** (`tools/scout/config.mjs` 参照):
- スキップワード: イラスト, デザイナー, ロゴ, 漫画, データ入力, ライター, 記事作成, アンケート 等
- マッチキーワード: AI, GAS, ChatGPT, LLM, コンサル, DX, 自動化, API 等
- listitem の中身にスキップワードが含まれる案件は除外
- 佐藤が指定した件数（デフォルト10件）になるまで収集
- ページ内で足りなければページ送り（後述）で追加取得

### Step 1-3: 各案件の URL を取得

snapshot の `link` ロール要素（案件タイトル名を持つもの）の href を `browser_get_attribute` で取得する。

```
for each job in filtered_jobs:
  browser_get_attribute ref={title_link_ref} name="href"
  → "/public/jobs/{job_id}" が返る
  → https://crowdworks.jp/public/jobs/{job_id} に組み立て
```

**注意点:**
- heading (h3) の ref ではなく、`role: link` でジョブタイトル名を持つ要素の ref を使う
  （heading は href を持たない。link 要素は snapshot 上部のインタラクティブ要素一覧にある）
- 一部の要素で `browser_get_attribute` がエラーになることがある。失敗したらリトライせず、別の案件で補充する
- クリック方式（click → snapshot → back）は使わない（遅い＋ heading にインターセプトされる問題がある）

### Step 1-4: 全タブ一括オープン

取得した URL を **新規タブ** で開く:

```
for each url in job_urls:
  browser_navigate url={url} newTab=true
```

これで全案件が別タブで開き、佐藤がタブを切り替えて自由に閲覧できる状態になる。

### Step 1-5: 一覧の提示

タブ番号と案件の対応表を佐藤に提示:

```
## CrowdWorks 案件一覧（10件）— 全てブラウザのタブで開いています

1. [案件タイトル] — 報酬: XX円 / 残りN日
2. [案件タイトル] — 報酬: XX円 / 残りN日
3. ...
...
10. [案件タイトル] — 報酬: XX円 / 残りN日

タブで各案件を確認して、応募したい番号を教えてください。
```

---

## Phase 2: 応募ループ（ユーザー駆動・何度でも繰り返し）

### Step 2-1: 待機

佐藤の指示を待つ。想定される入力:
- 「3番に応募」「1番」 → Step 2-2 へ
- 「もう終わり」「以上」 → ループ終了

### Step 2-2: タブ切替 + 詳細読取

1. `browser_tabs` action=`select` で該当タブに切替
2. `browser_snapshot` で案件詳細を取得
3. 以下の情報を抽出:
   - 案件タイトル
   - クライアント名
   - 報酬
   - 作業内容（全文）
   - 必要スキル
   - 応募期限
   - job_id（URL の末尾の数字部分）

### Step 2-3: 提案文の生成

案件詳細を元に、佐藤のプロフィール・スタイルに合わせた提案文を生成する。

**佐藤のプロフィール:**
- 業務自動化・AI活用のコンサルタントとして独立して活動
- 得意分野: GAS, API連携, ChatGPT/Claude活用, 業務自動化, DX支援
- 実績: 業務自動化システムの構築多数（予約管理, 顧客管理, メール自動送信, データ集計等）
- 強み: 「現場の業務フローを理解した上での最適な自動化設計」
- 代替手段の選定も得意（API/RPA/Webhook/スクレイピング）

**提案文の構成** (既存テンプレート `tools/scout/apply-message-4852692.txt` を参考):

```
はじめまして、佐藤と申します。
[案件名]の募集を拝見し、ご提案いたします。

【ご提案の概要】
（案件の課題を理解した上で、解決アプローチを1-2行で）

【実装アプローチ】
（具体的な技術・手順を3-5項目で。案件の要件に合わせる）

【想定スケジュール】
（Week単位で。案件の規模に合わせる）

【私の強み】
業務自動化・AI活用のコンサルタントとして独立して活動しております。
GAS・API連携による業務自動化の構築実績が多数あり、
特に「現場の業務フローを理解した上での最適な自動化設計」を得意としています。

まずは御社の状況をヒアリングさせていただき、
最適な実装方針をご提案できればと思います。
ご検討よろしくお願いいたします。
```

**提案文の提示:**

```
## 提案文（案件: [タイトル]）

[提案文の全文]

---
この内容で提案しますか？ 修正があれば教えてください。
```

佐藤が「OK」「いいね」「送って」等と言うまで、修正対応する。

### Step 2-4: 応募フォームへの入力

佐藤が承認したら:

1. 応募フォームに直接遷移:
   ```
   browser_navigate → https://crowdworks.jp/proposals/new?job_offer_id={job_id}
   ```

2. `browser_snapshot` でフォームの ref を取得（1回だけ）

3. **ログイン確認**: snapshot にログインフォームや「ログイン」リンクが表示されている場合、未ログイン:
   > CrowdWorks にログインが必要です。ブラウザで https://crowdworks.jp/login にアクセスしてログインしてください。完了したら教えてください。
   
   ログインは手動（自動ログインはしない）。同一ブラウザのタブは Cookie を共有するため、1タブでログインすれば全タブに適用される。
   ログイン後、同じ応募フォーム URL に再遷移して続行。

4. `browser_fill_form` で全フィールドを一括入力:
   ```
   browser_fill_form:
     fields:
       - element: "契約金額"
         ref: {金額textboxのref}
         value: "{金額}"
       - element: "完了予定日（年）"
         ref: {年comboboxのref}
         value: "{年}"
       - element: "完了予定日（月）"
         ref: {月comboboxのref}
         value: "{月}"
       - element: "完了予定日（日）"
         ref: {日comboboxのref}
         value: "{日}"
       - element: "応募有効期間"
         ref: {有効期間comboboxのref}
         value: "14"
       - element: "メッセージ"
         ref: {メッセージtextboxのref}
         value: "{提案文}"
   ```

   典型的なフォーム構造（ref は毎回 snapshot で確認）:

   - 契約金額（税抜）: textbox
   - 完了予定日（年/月/日）: combobox x3
   - 応募有効期間: combobox
   - メッセージ: textbox (required)

   金額のデフォルト: 案件の募集金額に合わせる。
   完了予定日のデフォルト: 応募日の1ヶ月後。
   応募有効期間のデフォルト: `14`（2週間）。

5. 入力完了後、佐藤に報告:

   > 応募フォームに入力しました。ブラウザで内容を確認して、送信ボタンを押してください。

   **送信ボタンはエージェントが押さない。** 佐藤が自分で確認して送信する。

### Step 2-5: 完了 → 待機

佐藤が「送った」「完了」「次」等と言ったら:

> 了解です。次の応募したい案件を教えてください。
> （残り: 2, 4, 5, 7, 8, 9, 10番）

応募済み番号を管理し、残りの未応募案件を表示する。
→ Step 2-1 に戻る。

---

## ページ送り（Phase 1 で使用）

検索結果が1ページに収まらない場合、または指定件数に満たない場合:

1. snapshot でページネーション要素（「次のページ」listitem 等）を探す
2. `browser_click` で次ページに遷移
3. 追加の案件を抽出・フィルタ
4. 必要件数に達するまで繰り返す

## 注意事項

- 佐藤が判断するまで自動応募しない（既存スカウトスキルと同じポリシー）
- **送信ボタンは絶対にエージェントが押さない**
- Phase 1 は最初の1回だけ実行。応募ループ中に再検索しない
- 佐藤が「もう終わり」「以上」等と言ったらループ終了
- Cookie は約2週間で失効する。Phase 2 中にログアウトされたらタブ上で再ログインを依頼
- CrowdWorks のDOM構造は変更される可能性がある。snapshot の結果が想定と異なる場合はスクリーンショットを撮って佐藤に状況を伝える
